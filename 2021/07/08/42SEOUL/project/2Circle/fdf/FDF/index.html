2CIRCLE - FDF

2021-07-08- 2021-07-15
4️⃣2️⃣ 42SEOUL-42SEOULFDF2CIRCLE

Abstract

  • mlx 사용 연습을 위한 프로젝트 (cub3D, miniRT)
  • 주어지는 파일(.fdf)을 3D 모델링/렌더링 하기
  • 사람에게만 평가 진행
  • 실행 파일은 fdf로 지정
  • Makefile 생성 및 기본 rule 포함
  • 기존 프로젝트인 libft를 clone하여 사용
  • norm(v3)에 맞춰서 구현
  • Segmentation fault, bus error, double free 등의 에러 발생하지 않도록 코드 작성
  • 허용 함수: open, read, write, close, malloc, free, perror, strerror, exit

Mandatory

  • (x, y, x)의 값들을 연결하여 굴곡진 지형을 표현한느 간단한 그래픽 “와이어프레임”모델 구현하기
  • 가로 인덱스는 가로축 위치에 / 세로 인덱스는 세로축 위치에 / 숫자값은 좌표상 점의 높낮이에 대응
  • 실행 시 다음과 같이 나오도록 만들기

Bonus

  • 구현 시 함수 제한없이 모두 사용 가능. 다만 동료평가 시 사용 이유에 대해 명확하고 타당하게 설명해야 할 것
  • subject에서는 보너스 예시를 보여줌. 즉 원하는 기능이 있다면 원하는대로 추가:
    • parallel 또는 conic projection 구현
    • zoom 기능
    • 이동 기능
    • 회전 기능

핵심 풀이과정

Bresenham Algorithm (브레즌햄 알고리즘)

  • 컴퓨터 그래픽에서 기본적으로 두 개의 점을 잇는 하나의 직선을 그리기 위한 효율적인 알고리즘
  • 정수 연산만을 이용하여 직선을 화면에 표현할 수 있기에 매우 효율적인 알고리즘

직선의 방정식

bresenham 알고리즘을 알기 위해서는 먼저 직선의 방정식을 알아야 한다.
먼저 두 점을 지나는 직선의 방정식은 다음과 같다.

y=(x2−x1)(y2−y1)∗(x−x1)+y1=mx+b{y} = \frac{(x2 - x1)}{(y2 - y1)} * {(x - x1)} + {y1} = mx + by=(y2−y1)(x2−x1)​∗(x−x1)+y1=mx+b

이러한 직선의 방정식은 그래픽스(특히 게임 프로그래밍)에서 많이 사용하는 수학공식이다.

예를 들면 쿼터뷰(마름모 모양 뷰) 게임에서는 캐릭터나 오브젝트가 서 있는 위치를 판별할 때 마름모의 각 선분(직선)으로 캐릭터위 위치를 쉽게 파악할 수 있다.

그렇다. 우리는 눈으로 캐릭터의 위치가 직선 위에 있는지 바로 확인할 수 있지만,
바보상자 컴퓨터는 직선(의 방정식)으로만 그 위치를 판별할 수 있다.


알고리즘 원리

먼저 기울기가 0과 1 사이인 직선을 가정해준다.
그리고 현재 좌표를 k번째라고 했을 때 좌표는 (xk,yk)(xk, yk)(xk,yk)라고 가정해보자.
이 다음에 찍어야 할 좌표값을 구해야 하는데, 우리는 아래의 가정을 갖고 좌표값을 구할 예정이다.

x -> 항상 1만큼 증가
y -> yk, yk+1 둘 중 하나 선택(기준: 중단점 mk + 1)
     - 중단점(mk + 1)이 임의로 설정한 직선 위에 위치: 그대로 yk
     - 중단점(mk + 1)이 임의로 설정한 직선 아래에 위치: yk + 1

여기서 중요한 점은 중단점 mk + 1이 직선 위 vs 아래 인지에 따라 다음 점을 판단하는 것인데, 이 판별식을 구해줘야 한다.

입력 받은 좌표 2개 (xl,yl),(xr,yr)(xl, yl), (xr, yr)(xl,yl),(xr,yr)로 분홍색 직선 하나 (xl,yl)−(xr,yr)(xl, yl) - (xr, yr)(xl,yl)−(xr,yr)를 만들어보자.

여기서 W=xr−xlW = xr - xlW=xr−xl , H=yr−ylH = yr - ylH=yr−yl이라는 값을 도출할 수 있다.

WWW와 HHH는 직선의 방정식에서 각각 (x2−x1)(x2 - x1)(x2−x1), (y2−y1)(y2 - y1)(y2−y1)에 대응된다.

대입 후 식을 간단하게 정리하면 −Wy+Hx+Wyl+Hxl<0-Wy + Hx + Wyl + Hxl < 0−Wy+Hx+Wyl+Hxl<0이라는 결과가 나오는데, 여기서 좀 더 정리하면
최종적으로 −2W(y−y1)+2H(x−Hxl)-2W(y - y1) + 2H(x - Hxl)−2W(y−y1)+2H(x−Hxl)이라는 식이 도출된다.
즉 이 식의 결과값이 0보다 작으면 직선의 위에, 0보다 크면 직선의 아래에 있다고 판별할 수 있다.

이런식으로 직선의 알고리즘을 이용하여 각 경우에 따른 식을 도출하면 된다.
(혹시나해서 적어두지만 알고리즘 설명은 기울기가 0과 1 사이일 때 가정하에서만 적어놨다. 즉 1 이상일 때의 판별식도 있다는 의미이다.
나머지 내용이 궁금한 사람은 위키를 참고하도록. 정말 잘 정리되어 있다)


알고리즘 적용하기

정리하자면 다음의 순서를 이용하면 된다.

- 직선의 시작점과 끝점을 입력받기
- 입력받은 시작점으로 W와 H를 구해 2H - W에 대입해 처음 초기값 F를 구한다
- F의 값이 0보다 크냐 작냐에 따라 판별식 F를 세우고, 점을 찍어준다.
- 기존 F에 각각 다른 공식을 F에 더해준다.
- 새로 나온 F로 직선 위인지 아래인지 판별식을 통해 점을 찍어준다(= 3번째와 동일함)

이런식으로 점을 계속 찍어나가면 직선을 만들 수 있다.
