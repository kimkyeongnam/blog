XSS

  • 서버가 아닌 사용자를 공격
  • 주로 게시판 댓글에 허용되는 HTML이나 JS 코드

SQL Injection

  • SQL 쿼리를 이용하여 시스템 접근 또는 로그인 우회
  • 쿼리문을 넣어 검색하면 접근하지 말아야 할 정보가 나올 수 있음

파일 업로드

  • PHP, JSP, ASP, 이미지, 영상 등을 업로드 하여 시스템 권한 획득 공격

디렉터리 노출

  • directory traversal attack

쿠키 또는 프록시 변조

쿠키

페이지 이동마다 Id, pw를 입력하지 않아도 자동으로 입력되는데, 이를 해킹하여 그 사람마냥 행동하는 것

  • TCP의 바탕이 되는 HTTP 프로토콜은 다른 프로토콜(FTP)과는 다르게 세션을 유지하지 않는 State-Less 방식을 이용하기 때문에 사용자의 세션 정보를 확인하기 위해서는 프로그램의 도움을 받아야 함
  • 프로그램에서는 쿠키나 세션을 이용하여 사용자를 추적할 수 있어, 페이지 이동 시마다 인증 정보 재입력 없이도 사용자를 구분할 수 있음
  • 그러나 이러한 프로그램적 접근 방식은 사용 방식에 따라 쿠키나 세션 변조 혹은 가로채기 등의 수법으로 인증 체계를 무너트릴 수 있는 위험성을 내포

프록시

서버에 전송되는 정보를 잠시 잡아서 원하는 내용으로 수정 후 서버에 전송(ex. 5000만원 쇼핑 후 1만원 결제)

  • 프록시 서버는 클라이언트와 서버 중간사이에 대리인 역할은 하는 서버, 이 프록시 설정은 본인의 로컬 호스트에도 설정이 가능함
  • 버프스윗이나 파로스를 이용해 조작 가능, 이러한 툴을 프록시 툴이라고 함. Request, Response 모두 가로채어 임의로 변조할 수 있음

에러노출과 구글해킹

구글 해킹 (Google Dork)

  • 검색을 통한 해킹이라고도 불립니다
  • 구글의 검색엔진이 시작된 세르게이 브린과 래리 페이지가 쓴 논문(The Anatomy of a Large-Scale Hypertextual Web Search Engine)을 살펴보면 PageRank를 통하여 ‘관련 있는 페이지가 수만 개라도, 그 중 최고의 웹 페이지만을 정확하게’ 찾아내게 알고리즘 되어 있음
  • 이러한 알고리즘을 실현시키기 위해서는 페이지 크롤링이 필수
  • 구글 크롤링은 Googlebot이 새로운 페이지 및 업데이트된 페이지를 찾아 Google 색인에 추가하는 과정 (이러한 크롤링 작업은 취약점 점검에도, 웹 공격에도 사용)
  • 이렇게 수집된 방대한 자료는 구글의 방식에 따라 색인됨
  • 여기서 색인된 수백억개의 페이지에서 취약한 페이지를 검색하는 것
