본 포스트는 제주코딩베이스캠프 지원 하에 작성되었습니다.
해당 강의를보고싶으신 분들은 여기를 참고하시길 바랍니다.


시스템 보안

  • 컴퓨터와 시스템을 장악하기 위한 가장 유효한 공격 방법
  • 취약점 가이드를 확인하자

사례

  • 2010년 이란의 핵 시설을 공격 - 스턱스넷
  • 2014년 한국수력원자력 발전소 해킹 - 한글
  • 6개월 이상의 잠복기간

발생하는 일

  • 시스템 마비
  • 시스템 정보 탈취
  • 계좌 획득
  • 정보 위변조

사용 언어

주로 C, Python(특히 Python은 해커들의 언어라고 할 만큼 많이 악용되는 상황)

인사이트

remote exploit, vulnerable exploit

  • https://www.rapid7.com/
  • https://www.outpost9.com/exploits
  • https://packetstormsecurity.com/

네트워크 보안

  • DDoS, DoS(Teardrop Attack, LAND Attack, SYN Flooding Attack, smurf Attack, …), 스니핑, 스푸핑, 세션 하이제킹, Scanning Attack
  • 대규모 공격. 진짜 공격을 숨기기 위한 용도로도 사용

사용 언어 & 프로그램

  • 공격 코드: Python(Python Dos code)
  • 패킷분석 : Wireshark

어플리케이션 보안

  • 응용 소프트웨어의 보안 정책에서의 결함이나 시스템 개발에서의 눈에 띄지 않는 위약점들 같은, 코드의 생명주기 전체 과정을 아우르는 것
  • 애플리케이션 보안 가이드, 시큐어 코딩 가이드 준수

리버스 엔지니어링

  • 역공학(逆工學)
  • 분해, 분석 -> 재조합, 새로운 제품
  • 취약점 분석과 악성코드 분석

포렌식

조사/수사, 파일 복구, 재현 등

사례

  • 황우석 박사의 줄기세포 사건 : 김선종 연구원의 노트북에서 포렌식을 통해 증거 획득, [유죄]
  • 2010년 총리실 민간인 불법 사찰 하드디스크 삭제 : 디가우저 장비를 이용, 하드디스크 데이터를 영구 삭제한 안티포렌식 사례
  • 군대에서도 파일삭제 시 30번 이상을 덮어씌우기해서 지움 (근데 그래도 복구되는 경우가 있다고 함)

증거

  • 직접증거, 간접증거, 인적증거, 물적증거
  • 파일만이 아니라 네트워크 상에 존재하는 증거, 메모리상에 존재하는 증거도 있음, 컴퓨터만 생각하고 쉬운데 서버 로그도 있음

5대 원칙

  • 정당성 원칙 : 입수된 증거가 적법한 절차와 방식으로 입수되었느냐
  • 재현의 원칙 : 직전과 같은 조건에서 현장 검증을 실시했을 경우 동일한 결과가 나오느냐
  • 절차 연속성의 원칙 : 획득, 이송, 분석, 보관. 보고서 작성 및 법정 제출의 각 단계에서 담당자와 책임자를 명확히하고 증거물의 변조, 손실에 대비하였느냐?
  • 무결성의 원칙 : 위조/변조되지 않았음을 입증할 수 있어야 한다
  • 신속성의 원칙 : 컴퓨터 시스템의 휘발성 정보수집 여부는 신속한 조치에 의해 결정되므로 모든 과정은 지체없이 진행되어야 함

팁

간단한 파일 복구가 필요 시 레쿠바 이용


버그헌팅

  • 취약점은 어디에나 존재
  • 완벽한 코드는 없다 = 풀리지 않는 자물쇠는 없다

하는 일

  • 소스코드감사 : 소스코드를 보며 취약점 찾기
  • 바이너리 감사 : 어셈블리어를 보며 취약점 찾기
  • 퍼징 : 자동으로 입력값을 넣어 취약점 찾기
  • 권한 상승
