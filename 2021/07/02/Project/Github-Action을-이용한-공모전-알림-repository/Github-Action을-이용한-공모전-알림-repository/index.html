Github Action을 이용한 공모전 알림 repository

2021-07-02
Project-crawlingpythongithubgithub action

해당 프로젝트 코드는 위 사이트에서 확인할 수 있습니다.

개요

자신의 실력을 향상시킴과 동시에 인정받고, 심지어 상금까지 받는다면 얼마나 좋을까?

새로운 인재를 찾기 위해, 기존의 관점을 180도 바꿔 사람들의 삶을 더욱 윤택하게 만들어줄 아이디어를 발견하기 위해 매일 수 많은 공모전 리스트가 쏟아져 나온다.

사실 공모전 사이트야 다양하다. 씽굿, 링커리어, 인크루트, 캠퍼스픽 등 다양한 곳에서 공모전 리스트를 보여준다.

하지만 사이트마다 장점과 단점이 있다.

  • 광범위한 카테고리: IT, UCC, 장학금 등 모든 것이 한 개로 묶여 있음
  • 세분화된 카테고리: 다양하지 않고 개수가 적은 공모전 리스트
  • 그외에 직접 사이트를 들어가서 일일이 확인해야하는 것

이러한 점에서 어느 사이트에 더 질 좋고 다양한 공모전이 있는지는 사용자가 일일이 접속해 찾아야 한다.

이 점에서 많은 시간이 아깝게 소요된다고 생각했고,
이를 해결하고자 3일에 한 번씩 오전 6시에 공모전 리스트를 업데이트하는 토이 프로젝트를 간단하게 진행해봤다.
카테고리는 내가 (예비)개발자이기 때문에 IT로 한정했다.

수행 추진 체계

  1. 공모전 사이트를 선정: 씽굿, 인크루트 공모전
  2. 각 사이트 크롤링: python3의 beautifulsoup과 selenium 이용
  3. 크롤링 결과에서 새로 등록된 공모전은 OPEN으로, 기존 Issue에서 기간이 마감된 공모전은 CLOSE로 변경
  4. 이 과정을 3일에 한 번씩 진행

사용 기술

  1. Crawling

    • python을 이용한 크롤링에는 두 가지 방법이 있다.

    • 먼저 Beautifulsoup은 정적페이지를 크롤링하는데 사용한다.

      $ pip3 install beautifulsoup4
      import requests
from bs4 import BeautifulSoup

    • selenium은 동적페이지를 크롤링할 때 사용한다.

      pip3 install selenium
      import selenium
from selenium import webdriver

    • 각 사이트에 맞춰 알맞은 크롤링 코드를 작성한다.

  2. Issue 설정

    • Github 계정 접근을 위해 github deploy key를 발급받는다.
    • 해당 키를 공개하게 된다면 엄청난 개인정보 소실로 이어지니 현재 사용 중인 repo에서 해당 키를 대체할 변수를 선언 및 값을 넣어준다.
    • 크롤링 결과물에서 새로 업데이트 된 내용들은 issue open으로 새로 생성,
      issue 목록에서 기한이 끝난 것들은 close로 설정한다.

  3. Github Action 설정

    • github/workflow를 사용하여 원하는 주기와 시간에 자동을 크롤링이 될 수 있도록 설정한다.

배운점 / 보안점

  • Github deploy key를 발급받아 repo에 설정해보는 경험
  • Github Action을 사용하기 위해 scheduling 공부. scheduling에서 주기설정에 대해 공부할 수 있었던 뜻깊은 시간이었다.
  • 구글링 실력 향상은 확실하게 된 것 같다. Github Action이 나온지 생각보다 별로 안되서 그런지 설명글이 많이 없더라. 강제로 영어해석해서 2시간만에 Github Action 뚝딱완성
  • 혼자하기 심심해서 42SEOUL 사람들 모아서 진행한건데, 코로나때문에 온라인으로 진행하다보니까… 내가 베이스 코드를 올려야 그제서야 하더라. 4명이 모여서 진행했지만 막상 90% 정도는 혼자 다했던, 좀 아쉬웠던 프로젝트
  • IT만 따로 분류된 카테고리가 존재하지 않았다. 일단 공학/IT계열 카테고리로 축소시키긴 했는데, 추후 업데이트 시 공모전 이름으로 IT인지 아닌지 판단하는 알고리즘도 만들어야 할 듯

여담

지난 블로그에서 해당 글을 작성할 때 간략하게 적어놔서 자세한 과정을 적지 못했는데,
블로그 글을 전부 옮겨온 이후, 다른 토이 프로젝트를 진행한다면 그 때부터는 자세히 과정을 적어둘 예정이다.
