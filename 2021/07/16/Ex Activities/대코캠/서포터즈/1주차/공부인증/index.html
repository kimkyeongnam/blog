Unix 운영체제

  1. Unix가 대부분 c언어로 작성 + 다른 컴퓨터에 수정해서 적용이 쉬움: 인기 급상승
  2. Unix를 변형할 수 있는 c언어가 핫이슈 됨
  3. Unix를 모방한 OS 출연 → 표준화를 위해 POSIX라는 규정을 만듦
    • Unix라면 갖춰야 할 규격과 기능. 만족시 Unix의 공식 버전으로 인정됨
  4. 왼쪽에 초록색 부분만 나눠져 있는 곳: Unix-like system에 Linux가 있음
    • AT&T에서 개발. AT&T의 소유물. 사용 시 여러 제약과 비용 발생(Unix 인증받을 때도 금전 소모)
    • 당시 컴퓨터업계에서 자유SW재단 활성화: 모두가 자유롭게 프로그램을 사용하자
    • 그곳에서 하는 운동이 GNU(GNU is Not Unix: GNU는 Unix가 아니다 개발자식 조크 미쳤다)만들기
      : 풀어 얘기하자면 Unix의 코드를 한 줄도 사용하지 않고 만든 Unix와 유사한 OS를 만들자
    • Linux는 POSIX는 만족하지만 공식 인증은 받지 않았기 때문에 Unix-like로 불림
    • 이러한 GNU + Linux kernal(Linus Torvalds가 개발한 커널)(kernel은 운영체제의 핵심 부분)
      = GNU/Linux… 지만 그냥 Linux로 부르는 사람이 많음
    • 최다 인원 프로젝트로 현재까지도 다양한 기업들에서도 기여하고 있음

세계 스마트폰의 82%, 임베디드 기계의 62%. 슈퍼컴퓨터의 99%가 Linux 사용

운영체제 저장장치 종류

종류 설명
ROM
(Read-Only Memory)
= 데이터 수정불가		 - 비휘발성
- 느리고 많은 용량(많은 용량일 수록 비쌈)
- 컴퓨터 기본 세팅 시 사용
RAM
(Random Access Memory)
= 데이터 수정가능		 - 휘발성
- 빠르고 작은 용량
- 컴퓨터 실행 시에만 사용
HDD - 하드디스크(Hard Disk Drive)
- 플래터가 원판을 읽어서 정보 저장: 움직이는 거리가 멀수록 쓰기 속도가 느려짐(병목현상)
- 물리적 충격에 약함
SSD - 반도체 메모리(플래시 메모리)
- 쓰기 속도가 동일
- 소음과 소비 전력도 적음
- 물리적 충격에 강함: 거의 모든 게이밍 노트북이 SSD로 갈아타는 추세
CPU cache
- 컴퓨터 성능 향상을 위해 존재하는 작고 매우 비싼 메모리
- 프로그램에서 직접적으로 읽거나 쓸 수 없음. 하드웨어의 메모리 관리 시스템(MMU)이 내부적으로 제어
- 성능향상법: 메인 메모리에 있는 데이터를 불러옴 → CPU가 필요한 데이터를 캐시에서 먼저 찾도록 함

register
- 다목적 저장 공간
- 데이터와 명령어를 저장하는 공간
- 일반적으로 현재 계산을 수행중인 값을 저장하는 데 사용
- 대부분의 현대 프로세서: 메인 메모리에서 데이터 옮겨옴 → 데이터처리 → 메인 메모리에 저장(로드-스토어 설계)
- 메모리 계층의 최상위에 위치. 가장 빠른 속도로 접근 가능한 메모리
